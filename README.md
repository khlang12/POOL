# POOL – Smart Contract-based Group Buying Platform  
*A Web3-powered collective purchasing experience without intermediaries.*

![POOL Project Overview](src/img/logo.png)
<br>

---

<br>

## 프로젝트 개요 (Overview)
**POOL**은  
기존 Web2 기반 굿즈 공동제작 및 공동구매에서 발생하는 구조적 문제를 해결하기 위해 기획된  
**스마트 컨트랙트 기반 Web3 공동구매 플랫폼**입니다.

총대를 맡는 개인 없이도,  
구매 희망자가 일정 수에 도달하면  
구매·결제·제작이 **자동으로 실행되는 구조**를 목표로 합니다.

- 프로젝트 기간: **2023.04 – 2023.05**  
- 프로젝트 성격: **GLITCH Hackathon 참여 프로젝트**  
- 형태: 팀 프로젝트  
- 대상 사용자: 공동구매 참여자 및 굿즈 제작자  

<br>

POOL is a Web3-based platform designed to address  
structural limitations in traditional group-buying systems.

By leveraging smart contracts,  
purchasing, payment, and execution are **automatically triggered**  
once participation thresholds are met.

- Project period: **Apr 2023 – May 2023**  
- Type: Hackathon team project  
- Target users: Group-buying participants and creators  

<br>

## 문제 정의 (Problem)

기존 공동구매 및 굿즈 제작 과정에서는  
다음과 같은 문제가 반복적으로 발생합니다.

- 총대(중개자)에 대한 **신뢰 부담**  
- 인원 미달 시 발생하는 **책임 불명확성**  
- 제작 및 정산 과정의 **불투명성**  

이러한 구조는  
개인에게 과도한 책임과 리스크를 집중시키는 방식이라고 판단했습니다.

<br>

In conventional Web2 group-buying systems,  
trust and responsibility are centralized in a single organizer.

This often leads to unclear accountability,  
inefficient processes, and a lack of transparency.

<br>

---

<br>

## 해결 방식 (Solution)

POOL은 공동구매를  
**신뢰에 의존하지 않는 구조적 시스템**으로 재설계하고자 했습니다.

스마트 컨트랙트를 통해  
조건 충족 여부와 실행 시점을 코드로 명확히 정의함으로써,  
사람이 개입하지 않아도 거래가 성립되는 구조를 제안합니다.

<br>

POOL approaches group purchasing as a **trustless, automated system**,  
where execution is governed by code rather than individuals.

<br>

### 주요 기능
- 공동구매 프로젝트 생성 및 참여  
- 참여 인원 충족 시 자동 실행  
- 스마트 컨트랙트 기반 조건 관리  
- 최초 도안 업로더에게 NFT 보상 제공  
- 중개자 없는 구조를 통한 책임 분산  

기능 확장보다는  
**문제 해결에 기여하는 구조 설계**에 집중했습니다.

<br>

### Key Features
- Creation and participation in group-buying projects  
- Automatic execution upon reaching participant thresholds  
- Smart contract-based condition and transaction management  
- NFT rewards for original design contributors  
- Intermediary-free purchasing flow  

The service prioritizes **structural clarity over feature quantity**.

## 기술적 접근 (Technical Approach)

- **React**  
  사용자 흐름과 상태 변화를 명확히 표현하기 위해 사용  

- **Blockchain / Smart Contract**  
  조건 충족 시 자동 실행되는 로직 구현  

- **Figma**  
  Web3 개념을 사용자 관점에서 이해 가능하도록 시각화  

<br>

기술 선택은 학습 목적이 아닌,  
**서비스 문제를 해결하는 데 적합한지**를 기준으로 결정했습니다.

<br>

- **React**  
  Applied to manage UI state and user flow clearly  

- **Blockchain / Smart Contract**  
  Used to encode execution logic and conditions directly into the system  

- **Figma**  
  Used to translate abstract Web3 concepts into user-facing flows  

<br>

All technical decisions were made based on  
**service requirements and problem-solving relevance**.
